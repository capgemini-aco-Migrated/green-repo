## Security Policy

### Supported Versions

The project is currently under development the supported versions
are considered release candidates.

| Version | Supported          |
| ------- | ------------------ |
| 0.0.x   | :white_check_mark: |

### Reporting a Vulnerability

> Don't file the vulnerability as an issue, follow the below process.

The capgemini-axis team and community take all security vulnerabilities in
bare-repository-template seriously. Thank you for improving the
security of bare-repository-template. We appreciate your efforts
and responsible disclosure and will make every effort to acknowledge your
contributions.

Report security bugs by emailing the lead maintainer at `yourmailaddress`.

The lead maintainer will acknowledge your email within 48 hours, and will send a
more detailed response within 48 hours indicating the next steps in handling
your report. After the initial reply to your report, the security team will
endeavor to keep you informed of the progress towards a fix and full
announcement, and may ask for additional information or guidance.

Report security vulnerabilities in third-party modules to the person or team maintaining
the module.

## Disclosure Policy

When the security team receives a security bug report, they will assign it to a
primary handler. This person will coordinate the fix and release process,
involving the following steps:

* Confirm the problem and determine the affected versions.
* Audit code to find any potential similar problem.
* Prepare fixes for all releases still under maintenance. These fixes will be
  released as fast as possible to npm.

## Comments on this Policy

If you have suggestions on how this process could be improved please submit a
pull request.
